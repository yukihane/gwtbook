= 古典的なGWTアプリケーション作成

本章ではGWT-WidgetとGWT-RPCを用いた「古典的な」GWTアプリケーションを作成してみます。
敢えて「古典的」と修飾しているのは、これらのパーツは新規アプリケーション作成時にはもはや使われるべきではないことを強調するためです。

ではなぜここで説明するのかというと、この古典的な方法はJavaプログラマフレンドリであり、
例えば短寿命で良いようなアプリケーション作成としては依然として利用しやすいと考えたからです。

「現代的な」GWTアプリケーションを作成するのであれば、本章は丸々スキップして構いません。
